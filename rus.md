Ваша цель — разработать **систему отзывов о продуктах**, которая предоставляет RESTful API. Компания предпочитает использование языка **Golang**, но вы можете выбрать любой другой язык.

Ниже приведены требования, разбитые по категориям:

### 1. Требования к API

Вам необходимо реализовать RESTful API со следующими возможностями:

* **API Продуктов (Products):**
* Действия: Создание, удаление, редактирование, получение списка и получение продукта по идентификатору.


* 
**Важное условие:** При запросе информации о продукте API **не** должен возвращать список отзывов, только средний рейтинг продукта.




* **API Отзывов (Reviews):**
* Действия: Создание, удаление и редактирование отзывов.


* Должны быть эндпоинты для просмотра отзывов к продукту.





### 2. Логика и функциональность

* **Расчет среднего рейтинга:**
* Система должна рассчитывать средний рейтинг продуктов.


* Этот расчет должен быть **потокобезопасным** (concurrency-safe).




* **Кэширование:**
* Отзывы о продуктах и средние рейтинги должны кэшироваться.




* **Событийная архитектура (Уведомления):**
* Сервис должен уведомлять другие сервисы при добавлении, изменении или удалении отзыва.


* 
**Демонстрация:** Вам нужно реализовать небольшой отдельный сервис, который будет просто логировать эти обновления, чтобы показать, что система уведомлений работает.





### 3. Структура данных

Вы можете использовать любые базы данных, но модели данных должны содержать следующее:

* 
**Product (Продукт):** Название, описание, цена, список отзывов и средний рейтинг (примечание: средний рейтинг не обязательно хранить прямо в модели продукта).


* 
**Review (Отзыв):** Имя, фамилия, текст отзыва и оценка (от 1 до 5).



### 4. Технический стек и настройка

* 
**Гибкость:** Вы можете использовать любые фреймворки, базы данных, механизмы кэширования и брокеры сообщений.


* 
**Окружение:** Проект должен запускаться через **Docker Compose**.


* 
**Качество кода:** Необходима конфигурация линтера.



### 5. Результат работы (Deliverables)

* 
**Код:** Публичный Git-репозиторий с кодом и конфигурационными файлами.


* 
**Документация:** Файл `README.md` или PDF с объяснением хода мыслей, принятых архитектурных решений и компромиссов (trade-offs).


* 
**Срок:** Результат нужно отправить на email в течение 5 рабочих дней.



**На что будут обращать внимание при проверке:**

* Проект должен быть легким в установке, расширяемым и поддерживаемым другими разработчиками.


* Обратите внимание на ваши **сообщения в коммитах** (commit messages).
